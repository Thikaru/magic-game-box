# Daily Games リポジトリ規約

このリポジトリは「毎日1つ新しいミニゲームが増える」静的サイトです。
GitHub Pages で公開され、Claude Code の Routines によって自動更新されます。
以下の規約は新規作成・修正のどちらのルーチンでも必ず守ること。

## ディレクトリ構成

```
/
├── index.html            # トップページ(本日のゲーム)※基本的に編集しない
├── archive.html          # 過去ゲーム一覧          ※基本的に編集しない
├── assets/style.css      # サイト共通スタイル      ※基本的に編集しない
├── games.json            # ゲームレジストリ(新規作成時に必ず追記)
└── games/
    └── YYYY-MM-DD-<slug>/   # ゲームごとに1フォルダ。フォルダ外に一切影響を与えない
        ├── index.html       # ゲーム本体(このファイルだけで完結してもよい)
        ├── game.js          # ロジック(分割する場合)
        └── meta.json        # メタ情報
```

## ゲームフォルダの命名

- `games/YYYY-MM-DD-<slug>/`。slug は英小文字とハイフンのみ(例: `2026-07-08-space-shooter`)
- 日付は実行日(Asia/Tokyo)

## games.json のスキーマ

新しいゲームを作ったら `games` 配列の**先頭**に追加する(先頭 = 最新 = 本日のゲーム)。

```json
{
  "games": [
    {
      "id": "2026-07-07-block-breaker",
      "title": "ブロックくずし",
      "genre": "アクション",
      "date": "2026-07-07",
      "description": "パドルでボールを打ち返してブロックを全部壊そう。",
      "path": "games/2026-07-07-block-breaker/"
    }
  ]
}
```

`meta.json` にも同じ内容を1オブジェクトで保存する。

## ゲームの必須要件

1. **説明から開始**: ロード直後はゲームを開始せず、タイトル・遊び方・操作方法を
   表示した説明画面(オーバーレイ)を出す。「スタート」ボタンで開始する。
2. **何度でもやり直せる**: ゲームオーバー/クリア画面に「もういちど」ボタンを置き、
   ページを再読み込みせずに最初から遊べるようにする。
3. **ヘッダーにトップへ戻るリンク**: `../../index.html` へのリンクを置く。
4. **完全に自己完結**: 外部ライブラリ・CDN・画像ファイルを使わない。
   Vanilla JS + Canvas または DOM のみ。絵は Canvas 描画・絵文字・CSS で表現する。
5. **PC とスマホの両対応**: キーボード操作に加え、タッチ/クリック操作を必ず実装する。
6. **1ファイル or 最小構成**: `index.html`(+必要なら `game.js`)に収める。

## ジャンルのローテーション

`games.json` を読み、直近7件と同じジャンルを避けて選ぶ。ジャンル例:
アクション / パズル / シューティング / レース / リズム / クイズ / タイピング /
反射神経 / 記憶 / カード / 育成 / ローグライク / スポーツ / 脱出 など。

## 品質チェック(コミット前に必ず)

- `python3 -m json.tool games.json` で JSON が壊れていないこと
- `node --check games/<id>/game.js`(game.js がある場合)で構文エラーがないこと
- 説明画面 → プレイ → 終了 → もういちど、の一連の流れが成立するコードであること

## Git 運用

- ルーチンは `claude/` プレフィックスのブランチにプッシュし、PR を作成する
- PR タイトルは新規作成なら `game: YYYY-MM-DD <タイトル>`、修正なら `fix: <対象ID> <概要>`
- `claude/**` ブランチの PR は GitHub Actions が自動マージする(.github/workflows/automerge.yml)
