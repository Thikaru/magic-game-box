# ルーチン①: 本日のゲームを新規作成

- 名前: `本日のゲーム作成`
- トリガー: Schedule — 毎日 6:00(タイムゾーンは Asia/Tokyo になっているか確認)
- リポジトリ: このリポジトリ
- コネクタ: 不要(全部外してよい)

以下をルーチンのプロンプト欄にそのまま貼り付ける。

---

あなたはこのリポジトリの「本日のゲーム」を作成する担当です。
まず CLAUDE.md を読み、規約をすべて守ってください。

手順:

1. `date +%Y-%m-%d` で今日の日付(Asia/Tokyo)を確認する。TZ=Asia/Tokyo を付けること。
2. `games.json` を読む。今日の日付のゲームがすでに存在する場合は何もせず終了する。
3. 直近7件のジャンルを確認し、それらと重複しないジャンルを1つ選ぶ。
   ジャンル内でも毎回ひとひねり入れ、過去作の焼き直しにしないこと。
4. `games/YYYY-MM-DD-<slug>/` フォルダを作成し、`index.html`(+必要なら `game.js`)と
   `meta.json` を実装する。CLAUDE.md の必須要件(説明画面から開始 / もういちどボタン /
   トップへ戻るリンク / 外部依存なし / タッチとキーボード両対応)を必ず満たすこと。
   見本として `games/2026-07-07-block-breaker/` の構造を参考にしてよい。
5. 難易度は「初見で1〜3分遊べて、少し悔しくてもう1回押したくなる」程度に調整する。
6. `games.json` の先頭に新しいゲームを追加する。既存エントリは変更しない。
7. 品質チェック:
   - `python3 -m json.tool games.json` が成功すること
   - `node --check` で JS の構文エラーがないこと
   - 開始→プレイ→終了→リトライの流れをコードレビューして論理的に確認すること
8. `claude/game-YYYY-MM-DD` ブランチにコミットし、
   PR タイトル `game: YYYY-MM-DD <ゲームタイトル>` で PR を作成する。
   PR 本文にはゲームの概要・操作方法・工夫した点を書く。

成功条件: 新しいゲームフォルダと games.json の更新だけを含む PR が1つ作成されていること。
ゲームフォルダの外(index.html / archive.html / assets / 過去のゲーム)は一切変更しないこと。
